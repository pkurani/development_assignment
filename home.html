issues

the point of this project is to essentially create an user-friendly, minimalist website that allows users to navigate/purchase products without being hampered with interface issues. in this section, we will be exploring how the design principles of usability, learnability, memorability, and accessability played a role in the structure and development of the site.

usability
  • to ensure usability, we had to ensure that all of the buttons worked and that all the edge cases were taken care of. includes ensuring that filter/sort options work, scrolling adapts to page without any issue, and users can add items to cart
  • another important consideration was for responsiveness. to ensure responsibivesnss to different screen sizes, flexboxes were used to adapt to the size of the page -- ensuring that users got the full experience.
  • furthermore, we also display a constantly updated list of products and total in the cart, which ensures that users have all the information they need about the products in their cart
learnability
  • learnability is accomplished by keeping the website fairly simple. by limiting the number of buttons that appear and labeling them approppriately, learnability does not present a significant issue for this website.
memorability
  • this is achieved by keeping a simple scanning pattern. the left side of the screen is dedicated to the products, while the left is dedicated to the carts.
  • on the left, each product card is displayed in an organized, uniform manner with spacing to "individualize" the cards.
  • in the cart, the product list is organized in a uniform manner as well. there is one card per row, thus allowing the cart to abidde by a simple "top to bottom" scanning pattern.
accessability
  • increased contrast between text and background to ensure that everything was extremely visible
  • ensured that page works if user needs to increase text size
takeaways
  • overall, building a website from react was a difficult and ardous process, but it made me more aware of interactions between components, and how the idea of design goes beyond what is just seen on screen
  • through this assignment, not only do i have a stronger understanding of the aforementioned processes, but i also believe i can build larger websites with more user interactions to truly create a more "holisitic" experience for a user
